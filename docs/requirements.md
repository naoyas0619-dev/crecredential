# クレクレデンシャル 要件定義書

## 1. 文書概要

### 1.1 文書の目的

本書は、資格学習記録アプリ「クレクレデンシャル」の開発に向けて、アプリの目的、対象ユーザー、機能要件、非機能要件、MVPの範囲を整理するための要件定義書である。

本書をもとに、後続工程で以下を作成する。

- ER図
- API一覧
- DBテーブル定義
- Spring Bootの実装方針
- AWSデプロイ構成

### 1.2 アプリ名

クレクレデンシャル

### 1.3 アプリ概要

クレクレデンシャルは、資格取得を目指すユーザーが、目標試験日から逆算した学習計画を作成し、日々の学習ログ、タスク、模擬試験結果を記録しながら、合格に向けた進捗を管理できるアプリケーションである。

初期開発ではフロントエンド画面は作成せず、Spring BootによるREST APIとして提供する。

## 2. 背景と目的

### 2.1 背景

資格学習では、試験日までに必要な学習量を把握し、継続的に学習することが重要である。一方で、学習計画の作成、日々の進捗管理、模擬試験結果の振り返りを個別に管理すると、全体の進捗が見えにくくなりやすい。

また、本アプリはSpring BootとAWSを学ぶための開発題材としても利用する。認証、CRUD、DB設計、Docker化、AWSデプロイなど、Webアプリケーション開発に必要な要素を段階的に学習できる構成とする。

### 2.2 目的

- 資格取得に向けた学習計画を管理できるようにする
- 学習タスクと学習ログを一元管理できるようにする
- 模擬試験結果を記録し、合格ラインとの差分を確認できるようにする
- 資格ごとの進捗状況を確認できるようにする
- Spring BootとAWSを用いたポートフォリオとして提示できる品質を目指す

## 3. 想定ユーザー

### 3.1 主な利用者

- IT資格や国家資格の取得を目指す個人学習者
- 学習計画を立てるのが苦手な人
- 勉強時間や模擬試験結果を記録して進捗を可視化したい人

### 3.2 間接的な閲覧者

- Spring Boot / AWSのポートフォリオとして確認する採用担当者
- 開発成果物をレビューするエンジニア

## 4. スコープ

### 4.1 MVPで対象とする範囲

初期リリースでは、以下の機能をREST APIとして提供する。

- ユーザー登録、ログイン、ログインユーザー取得
- 資格マスタの一覧取得
- ユーザーごとの資格目標登録、一覧取得、詳細取得
- 資格目標に紐づく学習計画の作成、取得
- 資格に紐づく教材一覧取得
- 学習タスクの作成、一覧取得、更新、完了
- 学習ログの作成、一覧取得
- 模擬試験結果の作成、一覧取得
- 資格目標ごとの進捗サマリー取得

### 4.2 MVPでは対象外とする範囲

- フロントエンド画面
- 外部APIによる教材の自動取得
- AIによる学習計画の自動生成
- メール通知
- SNSログイン
- 決済機能
- チーム、法人向け機能
- 画像アップロード
- 管理者画面
- 詳細なグラフ表示
- 本番レベルの監視、アラート

## 7. 非機能要件

### 7.1 技術構成

| 項目 | 内容 |
| --- | --- |
| 言語 | Java 21 |
| フレームワーク | Spring Boot 3系 |
| DB | PostgreSQL |
| 認証 | Spring Security + JWT |
| ORM | Spring Data JPA |
| DBマイグレーション | Flyway |
| API仕様確認 | Swagger / OpenAPI |
| ビルドツール | Gradle |
| フロントエンド | MVPでは対象外 |

### 7.2 AWS構成

| 項目 | 内容 |
| --- | --- |
| アプリ実行基盤 | ECS Fargate |
| コンテナイメージ管理 | ECR |
| データベース | RDS for PostgreSQL |
| ログ管理 | CloudWatch Logs |
| 権限管理 | IAM |
| ネットワーク | VPC, Security Group |

### 7.3 将来追加を検討するAWSサービス

| サービス | 用途 |
| --- | --- |
| S3 | ノート画像、添付ファイル保存 |
| Route 53 | 独自ドメイン管理 |
| ACM | HTTPS証明書管理 |
| Secrets Manager | DBパスワード、JWT秘密鍵などの管理 |
| GitHub ActionsまたはCodePipeline | 自動デプロイ |

### 7.4 セキュリティ要件

- パスワードは平文で保存しない
- 認証が必要なAPIではJWTを検証する
- ユーザーごとのデータ分離を徹底する
- 他ユーザーの資格目標、タスク、ログ、模擬試験結果にはアクセスできない
- DB接続情報やJWT秘密鍵は環境変数で管理する
- 本番環境では秘密情報をソースコードに含めない

### 7.5 運用・保守要件

- アプリケーションログをCloudWatch Logsで確認できる
- DBスキーマ変更はFlywayで管理する
- ローカル開発環境ではDocker ComposeでPostgreSQLを起動できる
- API仕様はSwagger / OpenAPIで確認できる

## 8. 初期データ方針

### 8.1 初期対象資格

MVPでは以下の資格を初期データとして登録する。

- AWS Certified Solutions Architect - Associate
- AWS Certified Developer - Associate
- Java Silver
- Java Gold
- 基本情報技術者
- 応用情報技術者

### 8.2 教材登録方針

- MVPでは教材情報を手動で登録する
- Webサイトは公式サイト、公式ドキュメントを優先する
- 書籍は直近数年以内のものを優先する
- 最近の書籍に良い候補がない場合は、定評のある書籍を登録する
- 教材の自動収集は将来拡張とする

## 9. 代表的なユースケース

### 9.1 資格学習を開始する

1. ユーザーがアカウントを登録する
2. ユーザーがログインする
3. ユーザーが資格一覧から取得したい資格を選ぶ
4. ユーザーが目標試験日、週の学習可能時間、現在レベルを登録する
5. ユーザーが資格目標に対する学習計画を作成する

### 9.2 日々の学習を記録する

1. ユーザーが学習タスクを作成する
2. ユーザーがタスクに沿って学習する
3. ユーザーが学習日、学習時間、学習内容、理解度を学習ログとして登録する
4. ユーザーが完了したタスクを完了状態にする

### 9.3 模擬試験結果を振り返る

1. ユーザーが模擬試験を受験する
2. ユーザーが試験名、受験日、得点、満点、合格ラインを登録する
3. ユーザーが苦手分野や復習メモを登録する
4. ユーザーが進捗サマリーで合格ラインとの差分を確認する

## 10. 制約事項

- 初期開発ではAPIのみを対象とし、画面は作成しない
- MVPでは外部APIによる教材取得は行わない
- MVPではAIによる学習計画生成は行わない
- MVPでは管理者画面を作成しないため、資格マスタと教材マスタは初期データとして登録する
- 本番運用相当の監視、アラートは初期対象外とする

## 12. 成功条件

MVPの成功条件は以下とする。

- ユーザー登録、ログインができる
- JWT認証が必要なAPIを保護できる
- ログインユーザーごとに資格目標、タスク、学習ログ、模擬試験結果を分離して管理できる
- 資格目標ごとの学習計画を取得できる
- 資格目標ごとの進捗サマリーを取得できる
- PostgreSQLを使ってデータを永続化できる
- Docker環境でアプリケーションを起動できる
- AWS ECS Fargate + RDS for PostgreSQLへのデプロイ方針を説明できる
